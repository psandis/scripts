# Infrastructure Scripts & Automation Notes

This repository contains a curated set of practical notes, command examples, and reusable snippets for infrastructure management, automation, and system administration.

It serves as a personal and professional reference for recurring tasks related to Ansible, Terraform, Docker, SSH, cron jobs, Linux utilities, and macOS scripting.

## Repository Structure

| File | Description |
|------|-------------|
| `ansible-scripts.md` | Ansible examples for automation, provisioning, and configuration management |
| `cron-and-automation.md` | Scheduled task examples, cron patterns, and simple automation workflows |
| `docker-scripts.md` | Docker command references, operational snippets, and container-related tasks |
| `macos-scripts.md` | macOS-specific commands and scripting helpers |
| `some-handy-scripts-linux.md` | General-purpose Linux scripts and useful command-line shortcuts |
| `ssh-and-remote.md` | SSH usage, remote connections, tunneling, and access management notes |
| `terraform-scripts.md` | Terraform commands, infrastructure workflows, and reusable IaC examples |
| `LICENSE` | License information for this repository |

## Why this repository exists

Instead of scattering useful commands across terminals, notes, and old projects, this repository keeps them documented in one place for faster reuse and easier maintenance.

It is designed to support:

- daily admin work
- infrastructure automation
- repeatable setup tasks
- troubleshooting
- command reference and learning

## Usage

Browse the individual `.md` files by topic and copy or adapt commands to your environment as needed.

Before using any script in production, verify:
- environment-specific values
- permissions and credentials
- target hosts and paths
- destructive or irreversible actions

## Disclaimer

The contents of this repository are intended as practical reference material.  
Use caution when running commands that modify systems, infrastructure, containers, or scheduled tasks.
